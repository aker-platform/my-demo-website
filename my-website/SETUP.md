# Investment Portfolio Setup Guide

This guide will help you complete the setup of Convex and Clerk for your investment portfolio application.

## 📋 Prerequisites

- Node.js installed
- A Convex account (free at https://convex.dev)
- A Clerk account (free at https://clerk.com)

## 🚀 Setup Steps

### 1. Convex Setup

1. **Create a Convex account** at https://convex.dev
2. **Run Convex development server:**
   ```bash
   npm run convex:dev
   ```
3. This will:
   - Open your browser to create/select a Convex project
   - Generate your `NEXT_PUBLIC_CONVEX_URL`
   - Automatically update `.env.local` with your Convex URL
   - Start watching for schema changes

4. **Deploy your schema and functions:**
   - The `convex:dev` command automatically deploys your schema
   - Your database tables will be created automatically

### 2. Clerk Setup

1. **Create a Clerk account** at https://dashboard.clerk.com
2. **Create a new application:**
   - Choose "Next.js" as your framework
   - Select authentication methods (Email, Google, GitHub, etc.)

3. **Copy your API keys:**
   - Go to "API Keys" in your Clerk dashboard
   - Copy the `Publishable key` and `Secret key`

4. **Update `.env.local`:**
   ```env
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
   CLERK_SECRET_KEY=sk_test_...
   ```

### 3. Connect Clerk with Convex

1. **In Clerk Dashboard:**
   - Go to "JWT Templates"
   - Create a new template called "convex"
   - Add this JSON:
     ```json
     {
       "aud": "convex",
       "sub": "{{user.id}}"
     }
     ```

2. **In Convex Dashboard:**
   - Go to Settings → Authentication
   - Select "Clerk"
   - Paste your Clerk Frontend API URL (found in Clerk settings)
   - Copy the Clerk Issuer URL from Clerk and paste it

3. **Save both configurations**

### 4. Run Your Application

1. **Start Convex (in one terminal):**
   ```bash
   npm run convex:dev
   ```

2. **Start Next.js (in another terminal):**
   ```bash
   npm run dev
   ```

3. **Open your browser:**
   - Navigate to http://localhost:3000
   - You should be redirected to sign in
   - Create an account or sign in


## 🔧 Configuration Files

### Environment Variables (`.env.local`)
- `NEXT_PUBLIC_CONVEX_URL`: Your Convex deployment URL
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`: Clerk public key
- `CLERK_SECRET_KEY`: Clerk secret key

### Convex Schema (`convex/schema.ts`)
Defines three tables:
- **portfolios**: User portfolio metadata
- **holdings**: Individual ETF positions
- **etfPrices**: Current and historical price data

### Clerk Integration
- `middleware.ts`: Protects routes requiring authentication
- `convex-client-provider.tsx`: Connects Clerk auth with Convex

## 🎯 Next Steps After Setup

1. **Seed initial data** (optional for testing):
   - Open Convex dashboard
   - Go to your deployment
   - Run the `etfPrices:seedPrices` mutation to add sample price data

2. **Customize authentication:**
   - Configure OAuth providers in Clerk
   - Customize sign-in/sign-up appearance
   - Set up webhooks for user events

3. **Integrate real ETF data:**
   - Use an API like Alpha Vantage, Polygon.io, or Yahoo Finance
   - Create Convex actions to fetch and update prices
   - Set up cron jobs for periodic updates

4. **Add features:**
   - Portfolio creation and management
   - Add/edit/delete holdings
   - Historical performance charts
   - Price alerts and notifications

## ⚠️ Troubleshooting

### Convex URL not found
- Make sure you've run `npm run convex:dev`
- Check that `.env.local` has `NEXT_PUBLIC_CONVEX_URL`
- Restart your Next.js dev server after adding the URL

### Clerk authentication not working
- Verify both Clerk keys are in `.env.local`
- Make sure JWT template is configured in Clerk
- Check that Clerk is connected to Convex in both dashboards

### Build errors
- Run `npm install` to ensure all dependencies are installed
- Check that all environment variables are set
- Verify TypeScript has no errors: `npm run build`
